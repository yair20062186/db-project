# מערכת ניהול מזון והסעדה לחטיבות מילואים – צה"ל
### IDF Food & Dining Hall Management System

**מגיש:** יאיר משה שושן  
**קורס:** מערכות בסיסי הנתונים – שלבים א', ב' ו-ג'  
**טכנולוגיות:** PostgreSQL (PL/pgSQL), Python  

---

## 📌 מבוא לפרויקט
פרויקט זה מציג מערכת בסיס נתונים רלציונית מלאה ומנורמלת מקצה לקצה (בדרגת נירמול 3NF) עבור ניהול מערך המזון, חדרי האוכל, הלוגיסטיקה והמלאי של יחידות המילואים בצה"ל. 

המערכת תוכננה כדי לתת מענה לניהול מורכב של חדרי אוכל בבסיסים שונים, מעקב מוקפד אחר מגבלות דיאטטיות ואלרגיות של לוחמים, ניהול מלאי דינמי עם תאריכי תפוגה, בקרה על ספקי המזון וניהול עלויות הארוחות.

---

## 📊 מבנה בסיס הנתונים (8 טבלאות)
המערכת מורכבת מ-8 טבלאות מרכזיות המקושרות ביניהן בקשרי גומלין ושלמות ישויות:

1. **SOLDIER (חיילים):** פרטים אישיים, מספר אישי, יחידה, דרגה, וסטטוס (סדיר/מילואים).
2. **DIETARY_RESTRICTION (מגבלות דיאטטיות):** מעקב רגישויות, אלרגיות (צמחוני, טבעוני, ללא גלוטן) ברמות חומרה שונות המקושרות לחייל.
3. **DINING_HALL (חדרי אוכל):** מיקומי חדרי האוכל בבסיסים השונים, קיבולת וסוג חדר האוכל.
4. **MEAL_SESSION (סבבי ארוחות):** תיעוד של כל ארוחה שהוגשה (בוקר, צהריים, ערב), כמות סועדים בפועל ועלות ארוחה.
5. **MENU_ITEM (פריטי תפריט):** קטלוג המזון, קלוריות, מחיר, הגדרות כשרות והתאמה לצמחונים/טבעונים.
6. **MEAL_MENU (תפריט הארוחה):** טבלת גשר (Junction Table) המקשרת בקשר של רבים לרבים (N:M) בין סבב ארוחה לפריטים שהוגשו בה.
7. **SUPPLIER (ספקי מזון):** ניהול ספקי המזון של הצבא, פרטי התקשרות וסטטוס פעילות.
8. **INVENTORY (ניהול מלאי):** מעקב כמויות מדויק של פריטי מזון בכל חדר אוכל בנפרד, כולל תאריכי תפוגה וזמני עדכון.

## 📈 היקף הנתונים במערכת (Data Volume)
בסיס הנתונים מאוכלס בלמעלה מ-**46,900 שורות** של נתונים הגיוניים ואותנטיים בעברית המדמים פעילות מבצעית חיה:
* **600** חיילים פעילים ביחידות השונות (גולני, צנחנים, שריון ועוד).
* **20,000** סבבי ארוחות מתועדים (`MEAL_SESSION`).
* **20,000** פריטים מנוהלים במלאי חדרי האוכל (`INVENTORY`).
